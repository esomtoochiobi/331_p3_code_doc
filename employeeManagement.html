employeeManagement module¶

employeeManagement.confirm_action()¶

This function will take the changes proposed in update_table(), and update the database accordingly, letting the user know if the action was successful or not.

employeeManagement.employee_data_getter()¶

This function accesses the database and returns a sorted employee list. The sorted part is important, otherwise other functions would break.

employeeManagement.show_employeeManagement()¶

This is the main function of the employee management page. Like other manager pages, it will restrict access unless the account is verified and has the right permissions. It uses employee_data_getter() in order to inject the employee table into the employeeManagement html file.

employeeManagement.submit_employee()¶

This will submit a new employee to the database, and then let the user know if it was successful.

employeeManagement.update_table()¶

This function is called when the user hits the ‘save’ button on the employee table, even if they do not make any changes - and it will construct a string to describe the differences between what is currently in the database and what they are changing it to. Additionally, it will tell the user if they are not making any changes. This function does NOT actually update the SQL table, but rather, asks the user if they are sure they want to commit the changes they have made. Actual changes made in confirm_action().
