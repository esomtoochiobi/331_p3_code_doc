trendsPage module¶

trendsPage.show_trendsPage()¶

Function: show_trendsPage Handles the route for displaying the trends page.

  • Retrieves user data from the session to validate permissions.
  • Redirects to the home page if the user is not logged in or is not a manager.
  • Connects to the database and sets up metadata reflection.
  • If the request is POST:
    • Fetches the selected start and end dates.
    • Queries the database to fetch:
      • Trends of menu items ordered together.
      • Popularity data of individual menu items.
    • Converts and formats the retrieved data for template rendering.
  • Renders the ‘trendsPage.html’ template with user data, date range, and trends/popularity data.
