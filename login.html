login module¶

login.customerLoginProcess()¶

Similar process to googleRedirect(), except instead of getting data from Google, data is retrieved from the database, and passed into session data.

login.googleLogin()¶

This is the first step in authenticating with Google. Sets up the uri that Google will redirect the user to after logging in, and dynamically will build it based on where it is hosted (locally or deployed). Also builds the url to redirect to Google, including the uri, and then redirects to that url. (Google OAUTH page)

login.googleRedirect()¶

This is where the user will land after being redirected by Google. It will harvest all of their essential user data (profile picture, name, email) and sell it to Facebook, as well as pass it into session data to be used on the site. The user’s permissions are set based on their email address. After this is done, the user is redirected to a login success page, which simply closes the window and the original window is reloaded, now with the user logged in.

login.logout()¶

Logging out simply clears the session and then spits you out at the home page.

login.show_customerLogin()¶

Shows customer login page (no Google OAUTH, just a customer lookup based on phone number).

login.show_login()¶

Shows the login html page with customer and employee sign in links. This is typically opened in a new window, which will be closed after a successful login.
